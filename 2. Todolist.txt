Concernant l'organisation des images (SEO):

Très important pour l’évolutivité et l’indexation SEO.

Nom descriptif : id-nom-produit-taille.ext

Exemples:
1234-chaussure-sport-red-thumb.webp
1234-chaussure-sport-red-medium.webp
1234-chaussure-sport-red-original.webp

Répertoire:
/public/images/products/
├── originals/       # images originales en haute résolution
├── thumbnails/      # petites versions optimisées pour les listes / cartes produits
├── medium/          # tailles intermédiaires (page produit)
└── placeholder/     # images génériques si un produit n’a pas d’image

---

Tu ne veux pas manuellement créer 3 versions de chaque image pour 10 000 produits.
➡ Solution : automatiser la génération de miniatures via un script ou un worker :

Lors de l’upload (dans ton back-office) :

Enregistrer l’image originale en haute résolution.

Générer les variantes (thumbnail, medium).

Optimiser (compression + WebP/AVIF si dispo).

Librairies utiles en PHP :

Intervention Image
 (super simple, stable)


---

use Intervention\Image\ImageManagerStatic as Image;

$img = Image::make($uploadedFile);

// Original
$img->save("images/products/originals/{$fileName}");

// Thumbnail (150x150)
$img->fit(150, 150)->save("images/products/thumbnails/{$fileName}");

// Medium (600x600)
$img->resize(600, 600, function ($constraint) {
    $constraint->aspectRatio();
    $constraint->upsize();
})->save("images/products/medium/{$fileName}");

---

4. Optimisation pour la performance

Formats modernes → WebP ou AVIF pour réduire le poids (tout en gardant un fallback JPEG/PNG si besoin).

Lazy Loading → pour les pages qui affichent beaucoup de produits :

<img src="/images/products/thumbnails/1234-chaussure-sport.webp" 
     loading="lazy" 
     alt="Chaussure de sport rouge">


Cela évite de charger 100 images d’un coup avant qu’elles soient visibles.

CDN si ton catalogue devient massif → les images seront servies depuis un serveur proche de l’utilisateur.

---

5. Évolutivité (100 000+ images)

À grande échelle, il est préférable d’éviter un seul dossier avec 100 000 fichiers → ça peut ralentir certains systèmes de fichiers.
Solution : arborescence hiérarchique basée sur l’ID :

/images/products/thumbnails/12/34/1234-chaussure-sport.webp


Ici, on découpe l’ID en sous-dossiers (12/34/) pour répartir la charge.
C’est ce que font les gros CMS comme Magento ou Shopify.
